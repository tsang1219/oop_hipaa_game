  <action>
    Create directory `attached_assets/generated_images/privacyquest/characters/`.

    Source character spritesheets from free/open-source sprite packs. Recommended packs to evaluate (all CC0 or CC-BY, chibi top-down RPG style):
    - **LPC (Liberated Pixel Cup)** characters from OpenGameArt.org — large community, many character variants, 64x64 frames that can be scaled down
    - **Kenney.nl** character packs — clean pixel art, CC0 license
    - **itch.io free sprite packs** — search "top-down RPG character spritesheet" filtered by free + CC0/CC-BY

    Selection criteria (per user decisions):
    1. Chibi / top-down RPG style (Zelda/Pokemon proportions — oversized head, compact body)
    2. Free/open-source license (CC0, CC-BY, or GPL)
    3. Must provide 4-direction walk cycles (down, up, left, right) with at least 3 frames per direction
    4. Consistent visual style across all characters (same proportions, outline style, color palette cohesion)
    5. Hospital-appropriate outfits or base sprites that can represent hospital roles through color

    For each of the 9 characters, select or customize a sprite from the pack(s):
    - **player**: HIPAA trainee — casual/business casual, visible badge or lanyard detail, distinct from all NPC types
    - **npc_receptionist**: Green scrubs or front-desk attire
    - **npc_nurse**: Blue scrubs, possibly with cap or stethoscope detail
    - **npc_doctor**: White lab coat over clothes
    - **npc_it_tech**: Dark polo/shirt, casual tech worker look
    - **npc_officer**: Dark uniform or suit (compliance officer)
    - **npc_boss**: Business suit, authoritative appearance
    - **npc_staff**: Orange/warm-colored scrubs or general hospital uniform
    - **npc_patient**: Light gray hospital gown
    - **npc_visitor**: Casual clothes, warm yellow/orange tones

    Diversity: Use different skin tones across the 9 characters (at minimum 3 distinct skin tones). If the pack provides skin tone variants, use them directly. If not, consider color-swapping skin pixels.

    Spritesheet format: Adapt to whatever the pack provides. Document the frame layout (rows x cols, frame size, direction order) in CREDITS.md so Plan 02 knows how to parse them. If the pack uses a format other than the standard "4 rows x 3 cols" (one row per direction, 3 frames each), note the exact layout.

    If frame size is not 32x32, that's fine — Phaser's `this.load.spritesheet()` accepts any `frameWidth`/`frameHeight` and the sprite will be rendered at whatever size is set. Document the actual frame dimensions.

    Save each character as a separate PNG spritesheet file named exactly as listed above.

    Create `CREDITS.md` in the characters directory with:
    - Pack name(s) and author(s)
    - License type and link
    - Spritesheet format documentation: frame size (WxH), grid layout (cols x rows), direction order (which row = which direction), frame order per row (idle, walk1, walk2)
    - Any modifications made (color swaps, scaling, compositing)
  </action>
