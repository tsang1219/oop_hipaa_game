**Fix 1 — onInteractZone: use functional setState form (Bug 2):**
Replace the stale-closure zone-completion block (around line 354) with the functional form:
```typescript
// Before (stale):
if (!completedZones.has(data.zoneId)) {
  const newZones = new Set(completedZones);
  newZones.add(data.zoneId);
  setCompletedZones(newZones);
}

// After (idempotent functional form):
setCompletedZones(prev => {
  if (prev.has(data.zoneId)) return prev;
  const next = new Set(prev);
  next.add(data.zoneId);
  return next;
});
```
Also apply the same functional pattern to the `onInteractItem` handler's `collectedItems` update (same file, around line 374) — identical stale-closure risk:

```typescript
// After (idempotent functional form):
setCollectedItems(prev => {
  if (prev.has(data.itemId)) return prev;
  const next = new Set(prev);
  next.add(data.itemId);
  return next;
});
```
After these changes, `completedZones` and `collectedItems` can be removed from the EventBridge effect's dependency array (they're no longer read from the closure — only written via functional form). Remove them from the dep array at line 399.
