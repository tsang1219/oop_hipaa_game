Three changes to PrivacyQuestPage.tsx:

**Change 1 — Module-level migration (before the component function, after imports):**
```typescript
import { migrateV1toV2, loadSave, writeSave, type SaveDataV2 } from '@/lib/saveData';
import roomDataJson from '@/data/roomData.json';

const ROOM_IDS = (roomDataJson.rooms as any[]).map((r: any) => r.id);
const initialSave = migrateV1toV2(ROOM_IDS);  // idempotent — returns v2 save
```
This runs during module evaluation, before React renders, so all useState initializers can read from `initialSave`.

**Change 2 — Replace 6 localStorage useState initializers with initialSave reads:**
Replace each direct `localStorage.getItem(...)` initializer:
- `completedRooms`: `useState<string[]>(initialSave.completedRooms)`
- `collectedStories`: `useState<string[]>(initialSave.collectedStories)`
- `completedNPCs`: `useState<Set<string>>(new Set(initialSave.completedNPCs))`
- `completedZones`: `useState<Set<string>>(new Set(initialSave.completedZones))`
- `collectedItems`: `useState<Set<string>>(new Set(initialSave.collectedItems))`
- `privacyScore`: `useState<number>(initialSave.privacyScore)`
- `gameStartTime`: `useState<number>(initialSave.gameStartTime)` (remove the lazy initializer entirely)
- `showIntroModal`: `useState(() => { if (qa-room check) return false; return !initialSave.onboardingSeen; })`
- `muted`: `useState<boolean>(initialSave.sfxMuted)`

**Change 3 — Replace 7 individual persistence useEffects with one consolidated writeSave():**
Remove these 7 effects (lines 138-144 and any similar ones):
```typescript
useEffect(() => { localStorage.setItem('gameStartTime', ...) }, [gameStartTime]);
useEffect(() => { localStorage.setItem('completedRooms', ...) }, [completedRooms]);
// ... etc (7 total)
```

Replace with a single effect that calls writeSave() with the full current state:
```typescript
useEffect(() => {
  writeSave({
    version: 2,
    completedRooms,
    collectedStories,
    completedNPCs: Array.from(completedNPCs),
    completedZones: Array.from(completedZones),
    collectedItems: Array.from(collectedItems),
    privacyScore,
    finalPrivacyScore,
    resolvedGates: Object.fromEntries(
      Array.from(Object.entries(resolvedGatesAll ?? {}))
    ),
    unlockedNpcs: Object.fromEntries(
      Array.from(Object.entries(unlockedNpcsAll ?? {}))
    ),
    npcPulsedRooms: npcPulsedRooms ?? [],
    gameStartTime,
    onboardingSeen: !showIntroModal,
    sfxMuted: muted,
    musicVolume: currentMusicVolume ?? 0.6,
  });
}, [completedRooms, collectedStories, completedNPCs, completedZones,
    collectedItems, privacyScore, finalPrivacyScore, gameStartTime,
    muted, showIntroModal]);
```

Note on resolvedGates/unlockedNpcs: The current page holds these as per-room `Set<string>` for the active room only. The v2 schema needs `Record<string, string[]>` (all rooms). For this stabilization phase, keep the per-room Set pattern but maintain a `resolvedGatesAll` ref that accumulates across room changes — update it whenever `resolvedGates` changes for `currentRoomId`. Same for `unlockedNpcs` / `unlockedNpcsAll`. Initialize from `initialSave.resolvedGates` and `initialSave.unlockedNpcs`.

If `resolvedGatesAll`/`unlockedNpcsAll`/`npcPulsedRooms` refs don't already exist in the file, add them:
```typescript
const resolvedGatesAll = useRef<Record<string, string[]>>(initialSave.resolvedGates);
const unlockedNpcsAll = useRef<Record<string, string[]>>(initialSave.unlockedNpcs);
const npcPulsedRooms = useRef<string[]>(initialSave.npcPulsedRooms);
```
And update `resolvedGatesAll.current[currentRoomId]` whenever the per-room `resolvedGates` state changes (add to the existing room-change effect or add a new small effect).
